## 1. 개요

**Safe Asset Mover**는 Unreal Engine 에디터에서 대량의 에셋을 **안전하게** 이동하기 위한 플러그인입니다.

### 핵심 기능

| 기능 | 설명 |
|------|------|
| **의존성 순서 이동** | 토폴로지 정렬로 참조 관계를 고려한 순서 자동 결정 |
| **레퍼런스/디펜던시 시각화** | 각 에셋의 참조 수·의존성 수 수치화 |
| **순환 의존성 경고** | 순환 참조 에셋을 주황색으로 강조 표시 |
| **P4 Checkout 감지** | 다른 사용자가 잠근 에셋을 이동 전에 식별 |
| **리다이렉터 정리** | FixUpRedirectors를 UI에서 직접 실행 |
| **다중 작업자 분업** | CSV 내보내기/가져오기 + 담당자 자동 배정 |
| **대상 경로 직접 편집** | Slate 테이블에서 행별 목표 경로 수정 |
| **Undo 지원** | 마지막 이동 세션 전체를 원래 경로로 복원 |

---

## 2. 설치

1. 프로젝트 루트의 `Plugins/` 폴더에 `SafeAssetMover` 폴더를 복사합니다.
2. `.uproject` 파일을 우클릭 → **Generate Visual Studio project files** 실행.
3. 솔루션을 열고 빌드합니다.
4. 언리얼 에디터 실행 후 **편집 → 플러그인** 메뉴에서 **Safe Asset Mover**가 활성화되어 있는지 확인합니다.

---

## 3. 도구 열기

상단 메뉴 **Tools → Safe Asset Mover**를 클릭합니다.

> 도구 창은 **Nomad Tab**으로 동작하므로 에디터 레이아웃 어디에나 도킹할 수 있습니다.

## 4. 기본 워크플로우

```
[스캔] → [대상 경로 확인/수정] → [이동] → [P4 Submit] → [FixUp Redirectors]
```

### 단계별 설명

#### 4-1. 소스·대상 폴더 지정

- **소스 폴더**: 이동할 에셋이 있는 Content Browser 경로 (예: `/Game/OldContent`)
- **대상 폴더**: 에셋을 이동할 목표 경로 (예: `/Game/NewContent`)
- 각 필드 오른쪽 `...` 버튼으로 폴더 선택 다이얼로그를 열 수 있습니다.

#### 4-2. 스캔

**스캔** 버튼을 클릭하면 소스 폴더 내 모든 에셋을 분석합니다.

- AssetRegistry를 통해 레퍼런스·디펜던시 수 계산
- 토폴로지 정렬로 **이동 순서(MoveOrder)** 결정
- 순환 의존성 감지 및 경고 표시

#### 4-3. 대상 경로 확인 및 수정

스캔 후 테이블의 **Target** 열에 자동 계산된 목표 경로가 표시됩니다.
셀을 직접 클릭하여 경로를 자유롭게 수정할 수 있습니다.

> **참고**: 배정이 완료된 경우(다중 작업자 모드), 내 담당 행만 편집 가능합니다.

#### 4-4. 에셋 선택

- **전체 선택 / 전체 해제** 버튼으로 일괄 처리
- 각 행의 체크박스로 개별 선택
- 순환 의존성(`⚠`) 에셋은 주황색으로 표시됨 — 이동 순서에 주의

#### 4-5. 이동 실행

**이동** 버튼을 클릭합니다.

- `IAssetTools::RenameAssets()` 를 사용하여 에셋 이동
- 이동 시 리다이렉터(Redirector)가 자동 생성됨
- 진행 상황은 프로그레스 바로 확인 가능
- 이동 중 **취소** 버튼으로 중단 가능

#### 4-6. P4 Submit

이동 완료 후 **Perforce에서 변경 사항을 Submit**합니다.

Submit이 완료되면 **"P4 Submit 완료"** 버튼을 클릭하여 확인합니다.
이 확인 없이는 **FixUpRedirectors**가 활성화되지 않습니다.

> 이 게이트는 Pending Changelist가 남아있는 상태에서 FixUp 실행 시 발생하는 P4 Checkout 오류를 방지합니다.

#### 4-7. FixUp Redirectors

**FixUp Redirectors** 버튼을 클릭하면 이동 과정에서 생성된 리다이렉터를 제거합니다.

## 6. 다중 작업자 협업 (CSV 분업)

여러 명이 에셋을 나누어 이동할 때 사용합니다.

### 6-1. 전체 흐름

```
[팀장: 스캔 후 CSV 내보내기]
        ↓
[팀장: 담당자 배정 실행]
        ↓
[팀장: CSV 내보내기 (배정 결과 포함)]
        ↓
[각 작업자: CSV 가져오기 → 내 담당만 이동]
```

### 6-2. 작업자 수 / 내 번호 설정

| 필드 | 설명 |
|------|------|
| **작업자 수** | 총 작업자 수 (기본값: 3) |
| **내 번호** | 내 작업자 번호 (1-based, 기본값: 1) |

### 6-3. 담당자 배정 (팀장)

1. 스캔을 완료합니다.
2. **작업자 수** 필드에 총 인원 수를 입력합니다.
3. **담당자 배정** 버튼을 클릭합니다.

**배정 알고리즘**:
- Union-Find로 의존 관계가 있는 에셋끼리 **같은 섬(Island)** 으로 묶음
- 같은 섬 = 반드시 같은 작업자에게 배정 (교차 이동 시 의존성 충돌 방지)
- 섬을 크기 내림차순으로 정렬 후 **Greedy Bin-Packing** 으로 부하 균등 배분

### 6-4. CSV 내보내기

**CSV 내보내기** 버튼 → 저장 위치 선택 → 파일 생성

CSV 형식:
```csv
SourcePath,TargetPath,AssigneeIndex
/Game/OldFolder/Tex_A.Tex_A,/Game/NewFolder/Tex_A,0
/Game/OldFolder/BP_B.BP_B,/Game/NewFolder/BP_B,1
```

| 열 | 설명 |
|----|------|
| `SourcePath` | 에셋 오브젝트 경로 (스캔 결과 기준) |
| `TargetPath` | 이동 목표 패키지 경로 |
| `AssigneeIndex` | 0-based 담당자 인덱스 (-1 = 미배정) |

> CSV 파일을 팀원에게 공유합니다.

### 6-5. CSV 가져오기 (각 작업자)

1. **내 번호** 필드에 자신의 번호를 입력합니다 (1-based).
2. **CSV 가져오기** 버튼 → 파일 선택.
3. 내 담당 에셋만 체크박스가 활성화되고, 나머지는 회색으로 비활성화됩니다.
4. **이동** 버튼으로 내 담당 에셋만 이동합니다.

> **주의**: CSV 가져오기 전에 반드시 **스캔**을 먼저 실행해야 합니다.
> CSV의 `SourcePath`가 스캔 결과와 일치해야 매칭됩니다.

### 6-6. 대상 경로 직접 수정

- 내 담당 행의 **Target** 열을 클릭하면 경로를 직접 편집할 수 있습니다.
- 편집 후 Enter 또는 포커스 이동으로 확정됩니다.
- 비담당 행은 회색으로 표시되며 편집 불가입니다.

### 6-7. 배정 로그 파일

담당자 배정 실행 시 아래 경로에 로그 파일이 자동 저장됩니다:

```
{프로젝트}/Saved/SafeAssetMover/Assignment_YYYYMMDD_HHMMSS.log
```

로그에는 각 섬의 배정 이유, 작업자별 부하, 에셋 목록이 포함됩니다:

```
[Distributor] 섬 #1  크기=15
[Distributor]   배정 이유:  작업자1:0  작업자2:0  작업자3:0  → 작업자1 선택 (부하 최소: 0 → 15)
[Distributor]   • Tex_Albedo
[Distributor]   • Tex_Normal
...
[Distributor] ⚠ 부하 불균형 감지: 최대=20, 최소=5, 평균=12.0
```

## 7. Perforce 연동

플러그인은 Perforce(P4)가 설정된 프로젝트에서 자동으로 연동됩니다.

### 동작 방식

| 시점 | 동작 |
|------|------|
| 이동 전 | P4 Checkout 시도; 다른 사용자가 잠근 경우 `LockedByOther` 상태로 건너뜀 |
| 이동 중 | 이동 성공 시 변경 사항이 P4 Pending Changelist에 추가됨 |
| FixUp 전 | **P4 Submit 완료** 확인 버튼 필수 (Pending CL 상태에서 FixUp 방지) |
| Undo 시 | P4 Pending 변경 사항 Revert 후 에셋 원위치 복원 |

### P4 잠금 에셋

- **상태** 열: `P4 잠금` (주황색)
- **상태** 열 툴팁: 잠금 중인 사용자 이름 표시
- **사유** 열: `P4 잠금: {사용자명}`
- 해당 에셋은 자동으로 건너뜀 (이동 실패로 카운트되지 않음)

---

## 8. Undo (되돌리기)

이동 완료 후 **Undo** 버튼으로 마지막 이동 세션 전체를 되돌릴 수 있습니다.

- 이동된 에셋을 원래 경로로 복원
- P4를 사용하는 경우 Pending Changelist를 Revert
- **FixUp Redirectors 실행 후에는 Undo가 제한**될 수 있습니다

> Undo 버튼은 이동 이력이 있을 때만 활성화됩니다.

## 9. 로그 및 진단

### 출력 로그 카테고리

에디터 **출력 로그** 창에서 `LogSafeAssetMover`로 필터링합니다.

| 레벨 | 내용 |
|------|------|
| `Log` | 일반 작업 진행 상황 |
| `Warning` | 파일 읽기 실패, 매칭 없음, 부하 불균형 등 |
| `Verbose` | CSV 행별 매칭 결과, 에셋 단위 배정 세부 내역 |

### CSV 가져오기 문제 진단

가져오기 실패 시 출력 로그에서 아래 패턴을 확인합니다:

```
[CSV] 파일 열기 시도: C:/path/to/file.csv        ← 경로 확인
[CSV] 파일이 존재하지 않음: ...                   ← 경로 오류
[CSV] 파일 읽기 실패 (잠금 또는 권한 문제일 수 있음) ← Excel 닫기
[CSV]   매칭 실패: '/Game/Old/Asset.Asset'       ← SourcePath 불일치
```

**체크리스트**:
1. 스캔을 먼저 실행했는가?
2. CSV 파일이 다른 프로그램(Excel 등)에서 열려있지 않은가?
3. CSV의 `SourcePath`가 오브젝트 경로 형식(`/Game/.../Asset.Asset`)인가?
4. CSV 파일의 인코딩이 UTF-8인가?

## 10. 열 참조

| 열 | 너비 | 설명 |
|----|------|------|
| ☑ | 고정 | 이동 대상 선택 체크박스 |
| **이름** | 가변 | 에셋 이름 (아이콘 포함). 순환 의존성 시 주황색, 비담당 시 회색 |
| **타입** | 고정 | 에셋 클래스 (Texture2D, Blueprint 등) |
| **레퍼** | 고정 | 이 에셋을 참조하는 에셋 수 (뱃지 색: 초록≤2 / 노랑≤9 / 빨강≥10) |
| **의존** | 고정 | 이 에셋이 의존하는 에셋 수 (동일 색상 규칙) |
| **순서** | 고정 | 토폴로지 정렬 기준 이동 순서 (낮을수록 먼저) |
| **상태** | 고정 | 현재 처리 상태 (→ [상태 참조](#11-상태-참조)) |
| **대상** | 가변 | 이동 목표 경로. 담당 행은 직접 편집 가능 |
| **사유** | 고정 | 이동 실패 시 원인 텍스트 |
| **담당** | 고정 | 배정된 작업자 번호 (미배정 시 `-`) |

## 12. 자주 묻는 질문

**Q. 스캔 후 에셋이 보이지 않습니다.**
A. 소스 폴더 경로가 `/Game/...` 형식의 콘텐츠 경로인지 확인하세요. 디스크 절대 경로는 지원하지 않습니다.

**Q. CSV 가져오기 후 아무것도 바뀌지 않았습니다.**
A. 스캔을 먼저 실행하고 CSV를 가져와야 합니다. 출력 로그의 `[CSV] 매칭 실패` 메시지로 어떤 경로가 불일치하는지 확인하세요.

**Q. FixUp Redirectors 버튼이 비활성화되어 있습니다.**
A. 이동 완료 후 **P4 Submit 완료** 버튼을 클릭해야 활성화됩니다. P4를 사용하지 않더라도 해당 버튼을 눌러야 합니다.

**Q. 순환 의존성 에셋도 이동할 수 있나요?**
A. 이동 자체는 가능합니다. 단, 순환 참조 에셋은 이동 순서가 보장되지 않으며, 같은 섬으로 묶여 반드시 같은 작업자에게 배정됩니다.

**Q. 담당자 배정 없이 이동할 수 있나요?**
A. 가능합니다. CSV 가져오기나 담당자 배정을 하지 않으면 모든 에셋의 `AssigneeIndex`가 -1(미배정) 상태로 유지되어 모든 에셋이 이동 가능 상태입니다.

**Q. Undo 후 리다이렉터가 남아있습니다.**
A. Undo는 에셋 경로만 복원합니다. FixUp Redirectors를 이미 실행한 경우 리다이렉터는 별도로 처리해야 합니다.
